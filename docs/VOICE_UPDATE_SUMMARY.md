# 发音人扩展更新说明

## 📋 更新概览

已成功将发音人从 **4 位**扩展到 **12 位**，新增 8 位不同风格的发音人。

## ✅ 已完成的修改

### 1. 后端更新 (ttsService.cjs)

#### VOICE_MAPPING 映射表
添加了 8 个新的发音人 ID 到 Edge TTS 名称的映射：
- `yunjian` → `zh-CN-YunjianNeural` (云健)
- `yunfeng` → `zh-CN-YunfengNeural` (云枫)
- `yunze` → `zh-CN-YunzeNeural` (云泽)
- `xiaochen` → `zh-CN-XiaochenNeural` (晓辰)
- `xiaomeng` → `zh-CN-XiaomengNeural` (晓梦)
- `xiaomo` → `zh-CN-XiaomoNeural` (晓墨)
- `xiaohan` → `zh-CN-XiaohanNeural` (晓涵)
- `xiaoxuan` → `zh-CN-XiaoxuanNeural` (晓萱)

#### getVoiceList() 函数
在发音人列表中添加了 8 个新的发音人对象，每个包含完整的信息：
- id, name, gender, language, description, originalId

### 2. 前端更新 (VoiceSelector.tsx)

在后备发音人数据数组中添加了相同的 8 个发音人对象，确保在 API 调用失败时也能显示完整列表。

## 📝 注意事项

1. **Edge TTS 依赖**：所有新增发音人都使用 Edge TTS (微软 Azure 神经语音)，需要网络连接。

2. **发音人 ID 一致性**：前后端的发音人 ID 必须保持一致，已确保所有 ID 匹配。

3. **后备数据**：前端的后备数据确保在 Electron API 调用失败时仍能显示发音人列表。

4. **热更新**：由于使用了 Vite HMR，更改已自动应用到正在运行的应用，无需重启。
