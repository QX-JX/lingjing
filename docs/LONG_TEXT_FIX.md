# 长文本试听问题修复

## 问题描述

用户反馈：**短文本可以试听，但长文本不能试听**

## 问题分析

经过代码分析，发现长文本处理存在以下问题：

### 1. **超时时间过短**
- 每个分段只有 5 分钟超时
- 对于长文本（>1000字），分段处理时总时间可能不够
- 没有根据文本长度动态调整超时时间

### 2. **错误处理不完善**
- 如果某个分段失败，整个流程就会失败
- 没有错误恢复机制
- 失败时临时文件可能残留

### 3. **缺少进度反馈**
- 用户不知道长文本正在分段处理
- 没有进度提示
- 用户可能误以为应用卡住了

### 4. **分段处理逻辑问题**
- 分段失败时没有清理临时文件
- 没有验证分段文件是否有效
- 错误信息不够详细

## 已实施的修复

### ✅ 1. 动态超时时间

根据文本长度动态调整超时时间：

```javascript
// 短文本（<500字）：2分钟
// 中等文本（500-1000字）：5分钟
// 长文本（>1000字）：10分钟
const textLength = safeText.replace(/<[^>]+>/g, '').length;
let timeout;
if (textLength < 500) {
    timeout = 2 * 60 * 1000; // 2 分钟
} else if (textLength < 1000) {
    timeout = 5 * 60 * 1000; // 5 分钟
} else {
    timeout = 10 * 60 * 1000; // 10 分钟
}
```

### ✅ 2. 错误恢复机制

添加了分段处理的错误恢复：

- **部分失败容忍**：如果失败的分段不超过 30%，继续处理剩余分段
- **自动清理**：失败的分段文件会自动清理
- **详细错误信息**：记录每个失败分段的原因

```javascript
// 如果失败的分段太多（超过30%），放弃整个任务
if (failedSegments.length > segments.length * 0.3) {
    // 清理已生成的分段文件
    segmentPaths.forEach(p => {
        try {
            if (fs.existsSync(p)) fs.unlinkSync(p);
        } catch (e) {}
    });
    throw new Error(`分段处理失败过多...`);
}
```

### ✅ 3. 进度日志和用户提示

**后端日志：**
```javascript
console.log(`[TTS] 文本过长 (${cleanTextLength} 字符)，进行分段处理`);
console.log(`[TTS] 已分成 ${segments.length} 段，开始逐段生成...`);
console.log(`[TTS] [${i + 1}/${segments.length}] 处理分段，长度: ${segmentLength} 字符`);
console.log(`[TTS] [${i + 1}/${segments.length}] 分段生成成功`);
console.log(`[TTS] 开始拼接 ${segmentPaths.length} 个音频分段...`);
```

**前端提示：**
- 长文本（>1000字）开始处理时显示提示
- 显示预计分段数量
- 超时错误时提供针对长文本的建议

### ✅ 4. 改进的文件验证和清理

- **文件验证**：生成后验证文件是否存在且不为空
- **自动清理**：失败时自动清理临时文件
- **拼接前检查**：确保有足够的分段可以拼接

```javascript
// 验证文件是否成功生成
if (fs.existsSync(segmentPath) && fs.statSync(segmentPath).size > 0) {
    segmentPaths.push(segmentPath);
    console.log(`[TTS] [${i + 1}/${segments.length}] 分段生成成功`);
} else {
    throw new Error(`分段 ${i + 1} 生成的音频文件无效`);
}
```

## 使用建议

### 对于长文本（>1000字）：

1. **耐心等待**
   - 长文本会自动分段处理
   - 每段可能需要 1-2 分钟
   - 总时间 = 段数 × 每段时间 + 拼接时间

2. **查看进度**
   - 打开开发者工具（F12）查看控制台
   - 可以看到每段的处理进度
   - 例如：`[TTS] [1/5] 处理分段，长度: 980 字符`

3. **如果超时**
   - 检查网络连接是否稳定
   - 尝试缩短文本长度（每次不超过 2000 字）
   - 分段生成后手动合并

4. **优化建议**
   - 单次生成不超过 2000 字（最佳体验）
   - 使用稳定的网络连接
   - 避开网络高峰期

## 技术细节

### 分段处理流程

```
长文本（>1000字）
    ↓
分段（每段最多 1000 字）
    ↓
逐段生成音频（带重试机制）
    ↓
验证每个分段文件
    ↓
拼接所有成功分段
    ↓
返回最终音频文件
```

### 错误处理策略

1. **网络错误**：自动重试 3 次
2. **分段失败**：记录错误，继续处理其他分段
3. **失败过多**：如果超过 30% 失败，放弃整个任务
4. **文件验证**：确保每个分段文件有效
5. **自动清理**：失败时清理临时文件
