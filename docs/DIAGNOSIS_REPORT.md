# 终端诊断报告

## 问题分析

通过检查终端日志，发现了以下问题：

### 1. **主要错误：NoAudioReceived**

所有 TTS 请求都返回 `NoAudioReceived` 错误，但网络连接测试通过。这说明：

- ✅ 网络连接本身正常
- ✅ edge-tts 模块工作正常
- ❌ 实际文本处理时出现问题

### 2. **文本编码问题**

日志中看到大量乱码字符（`�?`），可能原因：

- 文本中包含特殊字符或格式
- 文本编码问题（UTF-8 vs GBK）
- 文本传递过程中的编码丢失

### 3. **发音人分段处理**

日志显示：`[TTS] 使用发音人分段处理`

这说明文本包含发音人标记，会先按发音人分段，然后递归处理每个分段。

**潜在问题**：
- 递归调用时，如果分段文本很长（>1000字），应该会触发长文本分段
- 但可能在分段处理之前就因为网络问题失败

### 4. **文本长度检查**

日志中没有看到 `文本过长` 或 `分段处理` 的日志，说明：

- 要么文本长度没有超过 1000 字（去除标签后）
- 要么在到达长文本检查之前就失败了

### ✅ 3. **修复文本空白字符问题（2026-01-23）**

**问题根源**：从终端日志发现，发送给 Python 的文本包含前导换行符（如 `'\n\n"嗯，很大。"'`），导致 edge-tts 无法正确处理，返回 `NoAudioReceived` 错误。

**修复内容**：
1. **`generateAudioWithPython` 函数**：在文本赋值时添加 `.trim()`，确保发送给 Python 的文本已清理前后空白字符
2. **发音人分段处理**：在移除 voice 标签后，添加 `.trim()` 清理分段文本
3. **所有分段解析函数**：在 `parseVoiceSegments`、`parseSoundEffectSegments`、`parseRereadSegments`、`parseSpeedSegments` 中，确保存储的文本都经过 `.trim()` 处理

**修复位置**：
- `tauri-app/electron/ttsService.cjs` 第 85 行：`generateAudioWithPython` 函数
- `tauri-app/electron/ttsService.cjs` 第 838 行：发音人分段处理
- `tauri-app/electron/ttsService.cjs` 第 530、538、542 行：`parseSoundEffectSegments` 函数
- `tauri-app/electron/ttsService.cjs` 第 562、570、580、588 行：`parseVoiceSegments` 函数
- `tauri-app/electron/ttsService.cjs` 第 606、611、618、622 行：`parseSpeedSegments` 函数
- `tauri-app/electron/ttsService.cjs` 第 654、664、690、699 行：`parseRereadSegments` 函数

**预期效果**：所有发送给 edge-tts 的文本都会被清理掉前后的空白字符（包括换行符），避免 edge-tts 无法正确处理导致的 `NoAudioReceived` 错误。

### ✅ 4. **修复文本换行符规范化问题（2026-01-23）**

**问题根源**：从终端日志发现，即使文本前后空白字符已清理，文本中间的换行符（`\n`）仍然可能导致 edge-tts 处理失败。特别是多个连续的换行符可能引起问题。

**修复内容**：
1. **`generateAudio` 函数入口**：在函数开始时就清理文本前后的空白字符，确保所有递归调用都使用清理后的文本
2. **`preprocessSSMLTags` 函数**：添加换行符规范化处理：
   - 将3个或更多连续换行符替换为单个空格
   - 将2个连续换行符替换为单个空格
   - 将单个换行符替换为单个空格
   - 清理多个连续空格为单个空格
   - 清理前后的空白字符

**修复位置**：
- `tauri-app/electron/ttsService.cjs` 第 800-806 行：`generateAudio` 函数入口
- `tauri-app/electron/ttsService.cjs` 第 469-477 行：`preprocessSSMLTags` 函数

**预期效果**：所有文本在发送给 edge-tts 之前都会被规范化，换行符会被转换为空格，避免 edge-tts 无法正确处理导致的 `NoAudioReceived` 错误。
