# 历史备份功能说明

## 功能概述

历史备份功能可以自动保存您的编辑历史，方便您随时恢复之前的工作内容。系统会自动记录文本内容、发音人配置、音频参数等完整信息。

## 核心功能

### 1. 自动保存 ⏱️

- **自动保存间隔**：默认每 5 秒自动保存一次
- **智能防抖**：连续编辑时不会频繁保存，最小保存间隔为 3 秒
- **自动生成标题**：根据文本前 20 个字符自动生成记录标题
- **完整配置保存**：保存文本、发音人、语速、音调、音量、背景音乐等所有配置

### 2. 历史记录面板 📋

点击顶部导航栏的 **"历史记录"** 按钮即可打开历史记录面板。

#### 功能特性：

- **记录列表展示**：按时间倒序显示所有历史记录
- **智能时间显示**：
  - 今天的记录显示 "今天 HH:MM"
  - 昨天的记录显示 "昨天 HH:MM"
  - 更早的记录显示 "MM月DD日 HH:MM"
- **记录详情**：
  - 记录标题（文本前 20 字）
  - 保存时间
  - 字符数统计
  - 使用的发音人
  - 文本内容预览（最多 2 行）

### 3. 记录操作 🔧

#### 加载记录
- 点击任意历史记录即可恢复到编辑器
- 自动恢复文本内容和所有配置参数
- 加载后会显示成功提示

#### 删除记录
- 鼠标悬停在记录上时显示删除按钮
- 点击删除按钮并确认即可删除单条记录
- 删除操作不可恢复

#### 搜索记录
- 在搜索框中输入关键词
- 支持搜索记录标题和文本内容
- 实时过滤显示匹配结果

#### 清空全部
- 点击 "清空全部" 按钮
- 确认后清空所有历史记录
- 此操作不可恢复，请谨慎使用

### 4. 存储限制 💾

- **最大记录数**：100 条
- **超出处理**：自动删除最旧的记录
- **存储位置**：本地应用数据目录
- **数据格式**：JSON 文件

## 数据结构

```typescript
interface HistoryBackupRecord {
  id: string;                    // 唯一标识
  text: string;                  // 文本内容
  title: string;                 // 记录标题
  timestamp: number;             // 保存时间戳
  characterCount: number;        // 字符数
  voiceConfig: VoiceConfig;      // 发音人配置
  audioConfig: AudioConfig;      // 音频配置
  bgmConfig: BgmConfig;          // 背景音乐配置
}
```

## 使用建议

### 最佳实践 ✅

1. **定期查看历史**：养成定期查看历史记录的习惯
2. **及时清理**：定期删除不需要的记录，保持列表整洁
3. **重要内容备份**：对于重要内容，建议额外使用"导出项目"功能备份
4. **搜索快速定位**：记录较多时使用搜索功能快速找到目标

### 注意事项 ⚠️

1. **自动保存延迟**：自动保存有 5 秒延迟，快速切换时可能未保存
2. **删除不可恢复**：删除和清空操作不可恢复，操作前请确认
3. **存储限制**：最多保存 100 条记录，超出会自动删除旧记录
4. **本地存储**：数据存储在本地，重装系统或删除应用数据会丢失

## 故障排除

### 问题：历史记录无法保存

**解决方案：**
1. 检查应用是否有文件写入权限
2. 确认磁盘空间是否充足
3. 查看浏览器控制台是否有错误信息

### 问题：加载记录后配置未恢复

**解决方案：**
1. 确认该记录是否包含完整配置信息
2. 尝试重新保存并加载
3. 如果问题持续，请报告 Bug

### 问题：自动保存不工作

**解决方案：**
1. 检查文本内容是否发生变化
2. 确认至少等待 5 秒（自动保存延迟）
3. 查看控制台是否有错误信息
